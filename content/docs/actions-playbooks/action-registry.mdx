## Overview

The **action registry** lists actions Solvren can take. Actions can be read-only (recommend, notify) or write-back (update billing, fix pricing, etc.).

---

## Action Types

| Type | Description |
|------|-------------|
| **Recommend** | Suggest a fix; human executes |
| **Notify** | Alert stakeholders |
| **Simulate** | Run in dry-run mode |
| **Execute** | Apply changes via write-back (when policies allow) |

---

## Configuration

Actions are configured per playbook. Policies control when execution is allowed.
