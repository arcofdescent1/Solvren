## Overview

Issues are created when detector packs find problems in signals. The flow is:

1. Signals arrive and are stored
2. Detector packs run (scheduled or triggered)
3. Detectors evaluate signals against rules/models
4. When a problem is found, an issue is created
5. Impact engine estimates revenue at risk (if configured)
6. Issue appears in the ops inbox

---

## Issue Attributes

Each issue typically has:

- Source detector and signal
- Impact estimate
- Confidence score
- Recommended playbook(s)
