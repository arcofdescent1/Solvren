## Overview

An **issue** is a detected problem that may impact revenue. It is the core unit of work in the value loop.

---

## Issue Anatomy

| Field | Description |
|-------|-------------|
| **Source** | Detector and signal that produced the issue |
| **Impact** | Estimated revenue at risk |
| **Confidence** | How confident the system is in the finding |
| **Status** | Open, in progress, resolved, etc. |
| **Verification** | Whether the fix was verified and value attributed |
| **Recommended playbooks** | Actions that can address the issue |

---

## Lifecycle

Issues move through states: created → triaged → action taken → verified. Verification attributes recovered revenue or avoided loss.
