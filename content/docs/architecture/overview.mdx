## Overview

Solvren is built on Next.js with Supabase for data and auth. Key concepts include changes, approval workflows, evidence enforcement, and revenue impact reporting.

## Core Concepts

### Changes

A change represents a revenue-impacting modification (pricing, billing, integration, etc.). Each change has status, risk, and coordination metadata.

### Approval Workflow

Approvals flow from approval roles and mappings. The system suggests approvers and tracks decisions.

### Evidence

Evidence (screenshots, test results, docs) is required before submission. The platform enforces requirements by change type.

### Revenue Impact Report

The Revenue Impact Report summarizes exposure, risk drivers, and mitigations. It powers executive views and SLA tracking.

## Tech Stack

- **Frontend**: Next.js 16, React 19
- **Backend**: Next.js API routes, Supabase
- **Auth**: Supabase Auth
- **Database**: Supabase (PostgreSQL)
