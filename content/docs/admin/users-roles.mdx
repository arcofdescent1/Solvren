## Org roles

Solvren uses organization-level roles to define what users can do.

| Role | Responsibilities |
| --- | --- |
| Owner | Full configuration and governance control |
| Admin | Manage users, settings, and governance setup |
| Submitter | Create and submit changes |
| Reviewer | Review and approve changes |
| Viewer | Read-only access where allowed |

## Assigning roles well

Owner: executive sponsor. Admin: RevOps lead. Submitter: operators, engineers. Reviewer: finance, security stakeholders. Viewer: leadership, observers.

<Callout variant="tip" title="Good practice">
Keep the number of owners small. Use Admin for operational configuration.
</Callout>

## Related concepts

Org roles are different from approval roles, domain permissions, and explicit restricted-access grants.
