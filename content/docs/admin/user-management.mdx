## Overview

User Management allows administrators to control who can access Solvren and what actions they can perform.

Proper user configuration ensures that governance workflows involve the correct stakeholders.

---

## Inviting Users

Administrators can invite users to the organization.

To invite a user:

1. Navigate to **Admin → Users**
2. Click **Invite User**
3. Enter the user's email address
4. Assign a role

The user will receive an invitation email with instructions for joining the organization.

---

## User Roles

Roles determine what actions users can perform within the system.

Typical roles include:

### Owner

Full system access.

Owners can configure organization settings and manage governance policies.

---

### Admin

Administrators manage system configuration.

They can:

• invite users  
• configure approvals  
• manage evidence policies

---

### Submitter

Submitters can create and edit changes.

They can also submit changes for review.

---

### Reviewer

Reviewers evaluate and approve changes.

They provide feedback and determine whether governance requirements have been satisfied.

---

## Managing User Access

Administrators can update user roles at any time.

They can also:

• deactivate users  
• reassign roles  
• transfer ownership

Removing unused accounts helps maintain system security.

---

## Best Practices

Recommended practices include:

• assigning reviewers from each operational domain  
• limiting administrative privileges  
• removing inactive accounts promptly

These steps help ensure that governance workflows remain secure and efficient.
